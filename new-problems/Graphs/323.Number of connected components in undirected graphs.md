## 3) Mistakes / Confusions (Error Notebook 핵심)

- What I learned:

* union and find usage in terms of graphs
* Find(x): Returns the root (parent) of a node, helping us check which component the node belongs to (with path compression for speed).
* Union(x, y): Merges two different components by connecting their roots, reducing the total number of connected components.

---

## 4) Final approach (in plain English)

- Start by assuming every node is its own group.
- So if you have 5 nodes, you initially have 5 separate components.
- Go through each edge and check the two nodes it connects. Ask: “Are these two already in the same group or different groups?”
- If they’re in different groups, connect them (merge the groups).
- That means two components become one → so reduce the count by 1.
- If they’re already in the same group, ignore it.
- Because connecting them again doesn’t change anything.
- After processing all edges, whatever number you’re left with is the total number of connected components.
  Steps of the solution like you’d explain to a friend.

## 5) Code (final)

```python
class Solution:
    def countComponents(self, n: int, edges: list[list[int]]) -> int:
        parent=list(range(n))
        rank=[1]*n

        def find(x):
            if parent[x]!=x:
                parent[x]=find(parent[x])
            return parent[x]
        def union(x,y):
            rootX=find(x)
            rootY=find(y)

            if rootX== rootY:
                return 0
            if rank[rootX]>rank[rootY]:
                parent[rootY]=rootX
            elif rank[rootX]<rank[rootY]:
                parent[rootX]=rootY
            else:
                parent[rootY]=rootX
                rank[rootX]+=1
            return 1
        components=n

        for u,v in edges:
            components-=union(u,v)
        return components
```
