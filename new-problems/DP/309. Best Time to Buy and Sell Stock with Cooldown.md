## 1) Problem in 1 line

## 4) Final approach (in plain English)

- We move day by day and at each day decide whether to buy, sell, or skip.
- We keep track of whether we are currently allowed to buy or we need to sell.
- If buying, we either buy today (subtract price and switch state) or skip.
- If selling, we either sell today (add price and skip the next day for cooldown) or skip.
- We store results for each (day, state) in a hashmap to avoid recalculating the same decisions.
- We start from day 0 in buying mode and return the maximum profit possible.
  Steps of the solution like you’d explain to a friend.

## 5) Code (final)

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        #caching (index, buy/sell) ->tc/sc - O(n)

        #state: buying or selling?
        #if buy: i+1
        #if sell: i+2

        #caching done easily with hashmap
        dp={} #key=(i, buying) val =max_profit

        def dfs(i, buying):
            if i >=len(prices): #if index goes out of bounds
                return 0
            if (i, buying) in dp:
                return dp[(i, buying)]
            if buying:
                buy = dfs(i+1, not buying)-prices[i] #change the state, sell price-buy price at index = max profit
                cooldown = dfs(i+1, buying) #skipping the day
                #caching done here
                dp[(i, buying)]=max(buy, cooldown)
            else:
                sell=dfs(i+2, not buying) + prices[i]
                cooldown=dfs(i+1, buying)
                dp[(i, buying)]=max(sell, cooldown)
            return dp[(i, buying)]
        return dfs(0, True)
```
