## 1) Problem in 1 line

Buy exact `needs` of items at minimum cost using individual prices and unlimited special bundle offers.

---

## 2) My first thought (before coding)

Had no clue. Try all bundles greedily or sort offers by discount, but it fails because a bundle that looks best now can block a cheaper combination later.

---

## 3) Mistakes / Confusions (Error Notebook 핵심)

- Mistake #1: Forgot `self` in the method signature, so arguments shifted and Python tried indexing the `Solution` object (`TypeError: 'Solution' object is not subscriptable`).
- Mistake #2: Used the wrong recursive function name earlier (`shopping_offers` vs `shoppingOffers`) which would break recursion.
- What I learned:
- Memoisation - Memoization is storing the result of a solved subproblem so you don’t compute it again later. If the same state appears again, you return the stored answer instead of recalculating it.
- Related to knapsack problem so always draw the patters.
- Can solve this problem with lookup and not lookup as well(lookup table to not recalculate again again the solved sub problems).
- In LeetCode class methods, always include `self`, and memoize by “remaining needs” so repeated subproblems don’t explode.

---

## 4) Final approach (in plain English)

1. Treat the current remaining `needs` as a “state”.
2. Compute the baseline cost if we buy everything remaining at regular price.
3. For each special offer, check if it can fit (offer quantities ≤ remaining needs).
4. If it fits, apply it (subtract quantities) and recursively solve the smaller remaining needs, adding the offer price.
5. Memoize the best answer for each `needs` state so we never recompute it.

---

## 5) Code (final)

```python
class Solution:
    def shoppingOffers(self, price, special, needs, lookup=None):
        lookup = {} if lookup is None else lookup

        key = str(needs)
        if key in lookup:
            return lookup[key]

        # baseline: buy remaining items individually
        min_price = sum(needs[i] * price[i] for i in range(len(needs)))

        # try each offer if applicable
        for offer in special:
            if all(offer[i] <= needs[i] for i in range(len(needs))):
                new_needs = [needs[i] - offer[i] for i in range(len(needs))]
                min_price = min(
                    min_price,
                    offer[-1] + self.shoppingOffers(price, special, new_needs, lookup)
                )

        lookup[key] = min_price
        return min_price
```
