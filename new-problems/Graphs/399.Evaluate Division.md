# 399. Evaluate Division

**Difficulty:** Medium
**Tags/Pattern:** Graph, DFS, Backtracking
**Time/Space:** O(Q × (V + E)) / O(V + E)

---

## 1) Problem in 1 line

Given equations like `a / b = value`, determine the result of other division queries by using the relationships between variables.

---

## 2) My first thought (before coding)

My initial thought was to store the equations directly and try to answer queries from them.
But I realized some queries require combining multiple equations (like `a / c` using `a / b` and `b / c`), which means I need to treat the relationships like a graph and search through it.

---

## 3) Mistakes / Confusions (Error Notebook 핵심)

- Mistake #1: I was confused about how the equations translate into a graph. I learned that each variable is a node and each equation becomes a weighted edge between two nodes.
- Mistake #2: I learned how `defaultdict` works. It automatically creates missing keys, which makes it easy to build a graph without manually checking if a key exists.
- Mistake #3: I learned how `zip()` works. It lets us iterate through two lists (`equations` and `values`) at the same time so that each equation is paired with its corresponding value.
- What I learned: If I can find a path between two variables in the graph, multiplying the weights along the path gives the final division result.

---

## 4) Final approach (in plain English)

First, I build a graph from the equations.

For each equation like `a / b = 2.0`, I add:

- an edge from `a` to `b` with weight `2.0`
- an edge from `b` to `a` with weight `1 / 2.0`

Then for each query:

- If either variable does not exist in the graph, the answer is `-1.0`.
- If both variables are the same and exist in the graph, the answer is `1.0`.

Otherwise, I use DFS/backtracking to search from the starting variable to the target variable. While moving through the graph, I keep multiplying the weights of the edges. If I reach the target, the accumulated product is the answer. If no path exists, I return `-1.0`.

---

## 5) Code (final)

```python
from collections import defaultdict
from typing import List

class Solution:
    def calcEquation(self, equations: List[List[str]], values: List[float], queries: List[List[str]]) -> List[float]:

        graph = defaultdict(dict)

        def backtrack_evaluate(curr_node, target_node, acc_product, visited):
            visited.add(curr_node)
            ret = -1.0

            neighbors = graph[curr_node]

            if target_node in neighbors:
                ret = acc_product * neighbors[target_node]
            else:
                for neighbor, value in neighbors.items():
                    if neighbor in visited:
                        continue

                    ret = backtrack_evaluate(
                        neighbor,
                        target_node,
                        acc_product * value,
                        visited
                    )

                    if ret != -1.0:
                        break

            visited.remove(curr_node)
            return ret

        # Step 1: Build the graph
        for (dividend, divisor), value in zip(equations, values):
            graph[dividend][divisor] = value
            graph[divisor][dividend] = 1 / value

        # Step 2: Evaluate queries
        results = []

        for dividend, divisor in queries:
            if dividend not in graph or divisor not in graph:
                ret = -1.0
            elif dividend == divisor:
                ret = 1.0
            else:
                visited = set()
                ret = backtrack_evaluate(dividend, divisor, 1.0, visited)

            results.append(ret)

        return results
```
