## 1) Problem in 1 line

Build an array that can take snapshots over time and later return what any index’s value was at a specific snapshot id.

---

## 2) My first thought (before coding)

I first thought of saving a full copy of the array on every `snap()`, but that would be too slow and memory-heavy because snapshots can be many and the array can be large.

---

## 3) Mistakes / Confusions (Error Notebook 핵심)

- **Mistake #1:** Trying to store the whole array per snapshot (O(n) per snap) → leads to TLE/MLE.
- **Mistake #2:** Off-by-one confusion with `snap_id` (first `snap()` must return 0).
- **What I learned:** Don’t copy the whole array—store only changes, and treat `snap_id` as “current snapshot number” before incrementing.

---

## 4) Final approach (in plain English)

- For each index, keep a **history list** of pairs `(snap_id, value)` showing when that index changed.
- Start every index with `[(0, 0)]` so `get()` always has something to return.
- `set(i, val)`:
  - If we already set this index during the current snap_id, overwrite the last value.
  - Otherwise append a new `(snap_id, val)` entry.

- `snap()`: return the current `snap_id`, then increment it for the next snapshot.
- `get(i, snap_id)`: binary search the history for index `i` to find the last record with `record_snap_id <= snap_id`, and return its value.

---

## 5) Code (final)

```python
class SnapshotArray:

    def __init__(self, length: int):
        self.array=[[(0,0)] for _ in range(length)]
        self.snap_id=0


    def set(self, index: int, val: int) -> None:
        self.array[index].append((self.snap_id, val))


    def snap(self) -> int:
        self.snap_id+=1
        return self.snap_id-1


    def get(self, index: int, snap_id: int) -> int:
        history=self.array[index]
        left,right=0, len(history)-1
        while left<=right:
            mid=(left+right)//2
            if history[mid][0]<=snap_id:
                left=mid+1
            else:
                right=mid-1
        return history[right][1]
```
