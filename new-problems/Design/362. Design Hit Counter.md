## 4) Final approach (in plain English)

- Keep a queue of timestamps for every hit you receive.
- When a new hit happens, just add its timestamp to the back of the queue.
- When someone asks for getHits(t), first throw away old hits from the front that are <= t - 300.
- After cleanup, everything left in the queue happened within the last 5 minutes.
- Return the queue length as the answer.
  Steps of the solution like you’d explain to a friend.

## 5) Code (final)

```python
class HitCounter:

    def __init__(self):
        self.q=deque() #store timestamps of hits

    def hit(self, timestamp: int) -> None:
        self.q.append(timestamp) #record a hit


    def getHits(self, timestamp: int) -> int:
        #remove hits older than 300 seconds
        while self.q and self.q[0]<=timestamp-300:
            self.q.popleft()
        return len(self.q)
```
