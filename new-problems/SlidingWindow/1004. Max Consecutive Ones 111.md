## 1) Problem in 1 line

- Find the longest subarray of 1s you can get by flipping at most `k` zeros in the binary array.
  Explain the problem in one sentence.

## 2) My first thought (before coding)

## 3) Mistakes / Confusions (Error Notebook 핵심)

- What I learned:

* I learned that in sliding window problems, you expand the right pointer normally and only move the left pointer when the condition breaks.
* I understood that instead of recalculating everything, I can track the count of zeros inside the window.
* When zeros exceed `k`, I shrink the window from the left until the window becomes valid again.
* I realized that each element is visited at most twice, which makes the solution O(n).
* I learned that maintaining a running count is often enough to control the window correctly.

---

## 4) Final approach (in plain English)

- Start with two pointers, `left` and `right`, to represent a sliding window.
- Move the `right` pointer forward one step at a time, expanding the window.
- Keep track of how many zeros are inside the window.
- If the number of zeros becomes greater than `k`, move the `left` pointer forward until the window has at most `k` zeros again.
- At every step, calculate the window size (`right - left + 1`) and update the maximum length.
- Continue until the `right` pointer reaches the end, and return the maximum length found.

Steps of the solution like you’d explain to a friend.

---

## 5) Code (final)

```python
class Solution:
    def longestOnes(self, nums: List[int], k: int) -> int:
        left=0
        curr=0
        ans=0
        for right in range(len(nums)):
            if nums[right]==0:
                curr+=1
            while curr>k:
                if nums[left]==0:
                    curr-=1
                left+=1

            ans=max(ans, right-left+1)
        return ans
```
