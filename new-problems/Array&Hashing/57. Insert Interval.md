## 1) Problem in 1 line

- Insert a new interval into a sorted, non-overlapping list of intervals and merge any overlapping intervals so the final list remains sorted and non-overlapping.
  Explain the problem in one sentence.

---

## 2) My first thought (before coding)

- I first merged the existing intervals using the standard merge logic.
- Then I tried to adjust the merged list to include `newInterval`.
- I separated the process into two passes because I thought it would simplify handling overlaps.

What I tried first, and why.

---

## 3) Mistakes / Confusions (Error Notebook 핵심)

- Mistake #1: I tried to merge existing intervals first and then insert the new interval separately, which made the logic unnecessarily complicated.
- Mistake #2: I incorrectly indexed intervals (like using interval[-1][1]), not realizing each interval is just [start, end] and doesn’t need double indexing.
- What I learned:

* I can handle insertion and merging in **one pass** instead of separating them.
* Keeping a clear overlap condition simplifies the logic significantly.
* Avoid overcomplicating interval problems — follow the pattern: before overlap, merge overlap, after overlap.
* Clean indexing and understanding the interval structure prevents logical errors.

---

## 4) Final approach (in plain English)

- Go through the intervals from left to right.
- First, add all intervals that end before the new interval starts (no overlap).
- Then, merge all intervals that overlap with the new interval by expanding its start and end.
- Once merging is done, add the updated new interval to the result.
- Finally, add the remaining intervals that start after the new interval ends.
  Steps of the solution like you’d explain to a friend.

## 5) Code (final)

```python
class Solution:
    def insert(self, intervals: List[List[int]], newInterval: List[int]) -> List[List[int]]:
        n=len(intervals)
        i=0
        res=[]
        #work this out with and example - BETTER UNDERSTANDING
        #case1: no overlapping before merging intervals
        while i<n and intervals[i][1]<newInterval[0]:
            res.append(intervals[i])
            i+=1
        #case2: overlapping and merging intervals
        while i<n and newInterval[1]>=intervals[i][0]:
            newInterval[0]=min(newInterval[0], intervals[i][0])
            newInterval[1]=max(newInterval[1], intervals[i][1])
            i+=1
        res.append(newInterval)
        #case3: no overlapping after merging newInterval
        while i<n:
            res.append(intervals[i])
            i+=1
        return res
```
