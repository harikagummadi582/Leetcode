## 1) Problem in 1 line

- Given n nodes and a list of edges, determine whether the graph forms a valid tree (i.e., it is fully connected and contains no cycles).
  Explain the problem in one sentence.

## 3) Mistakes / Confusions (Error Notebook 핵심)

- What I learned:

* In undirected graphs, always skip the parent to avoid false cycle detection.
* DFS should return a boolean to explicitly detect cycles and must be checked.
* A valid tree = no cycles + fully connected + exactly n - 1 edges.
* Just traversal is not enough — need to validate graph properties carefully.

---

## 4) Final approach (in plain English)

- A valid tree must have n - 1 edges, otherwise return False.
- Convert edges into a graph (adjacency list).
- Use DFS to traverse the graph from node 0.
- If we visit a node twice (not via parent), a cycle exists → not a tree.
- After DFS, if all nodes are visited → graph is connected → valid tree.
  Steps of the solution like you’d explain to a friend.

## 5) Code (final)

```python
class Solution:
    def validTree(self, n: int, edges: List[List[int]]) -> bool:
        #does edges given make it a valid tree?
        #n is given and so are edges
        # Tree must have exactly n-1 edges
        if len(edges)!=n-1:
            return False
        graph={i:[] for i in range(n)}
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        #to track and detect cycles
        visited=set()

        def dfs(node,parent):
            if node in visited:
                return False #cycle found
            visited.add(node)

            for neighbor in graph[node]:
                if neighbor==parent: #to avoid unreal cycles
                    continue
                #if dfs fails then cycle detected so return false
                if not dfs(neighbor, node):
                    return False #this is internal false
            return True

        if not dfs(0,-1):
            return False

        return len(visited)==n
```
