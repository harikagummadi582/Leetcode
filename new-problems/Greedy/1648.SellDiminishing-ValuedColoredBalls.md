## 1) Problem in 1 line

- We have balls of different colors, and each time we sell a ball its value equals how many balls of that color are left, so we need to sell `orders` balls in a way that **maximizes total profit**.

---

## 2) My first thought (before coding)

My first instinct was to **always sell the ball with the highest value**. I thought about using a **max heap (priority queue)** where I repeatedly pick the largest inventory value, add it to the profit, decrease it by 1, and push it back. This works logically, but it becomes too slow when the number of orders is very large.

---

## 3) Mistakes / Confusions (Error Notebook 핵심)

- **What I learned:**
  Instead of selling balls individually, we can **sell entire value levels at once** and use the **arithmetic series formula** to calculate the total profit efficiently.

## 4) Final approach (in plain English)

**1. First, sort the inventory in descending order so we always deal with the highest valued balls first.**
This helps us focus on the most profitable balls first, because the customer always pays more for colors that currently have more balls left.

**2. Track how many colors currently share the same highest value using `k`.**
Sometimes more than one color can have the same count, like `[5,5,5,3]`.
So instead of treating them separately, we group them together and say: there are `k` stacks at this same level.

**3. Compare the current level (`cur`) with the next level (`nxt`) to see how many value levels we can drop.**
For example, if `cur = 5` and `nxt = 3`, we can sell values `5` and `4` before reaching the next smaller level.
This tells us how many balls we can sell in one batch.

**4. If we have enough orders, sell all balls from `cur` down to `nxt + 1` using the arithmetic sum formula.**
Instead of selling one by one, we directly calculate the sum of those values.
So if we sell `5 + 4`, or `5 + 4 + 3`, we use the formula for consecutive numbers to get the total quickly.

**5. If orders are not enough, sell only the required full levels and then sell the leftover balls separately.**
Example: if 3 stacks are at value `5` and we only have 4 orders, we can sell one full level (`5,5,5`) and then 1 extra ball at the next lower value (`4`).
After that, we stop because all orders are used.

**6. Keep adding profit throughout the process and return the final answer modulo (10^9 + 7).**
The modulo is only there because the profit can become very large, and the problem asks us to return the answer in that format.

## Super simple intuition

We are basically **selling the tallest stacks first**, level by level, instead of ball by ball.
That is what makes the solution fast and optimized.

## 5) Code (final)

```python
class Solution:
    def maxProfit(self, inventory: List[int], orders: int) -> int:
        MOD=10**9 + 7
        inventory.sort(reverse=True)
        inventory.append(0)

        ans=0 #profit
        k=1 #no of colors at highest level

        for i in range(len(inventory)-1):
            cur,nxt=inventory[i], inventory[i+1]

            if cur>nxt:
                balls=k*(cur-nxt)

                if orders>=balls:
                    ans+=k*(cur+nxt+1)*(cur-nxt)//2
                    orders-=balls
                else:
                    full=orders//k
                    rem=orders%k
                    low=cur-full

                    ans+=k*(cur+low+1)*full//2 #arthimetic operation
                    ans+=rem*low
                    return ans%MOD
            k+=1 #if multiple same levels with same number of balls
        return ans%MOD
```
